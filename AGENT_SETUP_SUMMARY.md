## What the Agent Does

### Phase 1: Research & Fact-Check (30-40 min)
- Reviews latest AI developments from the past 7 days
- Focuses on: agent orchestration, multi-agent systems, safety, evaluation
- Checks for framework deprecations and breaking changes
- Identifies accuracy issues in existing guides

### Phase 2: Update Repository (20-30 min)
**MANDATORY COMPLIANCE:** All changes must pass STYLE_GUIDE_CHECKER.md before committing
- Updates guides with current information
- Manages homepage news: keeps 10 latest, archives older ones
- Creates/updates `news-archive.html` when needed
- Verifies all code examples work with current framework versions
- Maintains consistency: fonts, colors, spacing, terminology

### Phase 3: Commit & Notify (10 min)
- Makes professional commits with zero AI attribution
- Sends push notification to Claude mobile
- Updates memory for continuity across weeks

## Compliance Standards

### Style Guide (11 Sections)
1. **HTML Structure** — Semantic markup, proper hierarchy
2. **Typography** — Serif (headers), Sans (body), Mono (code)
3. **Color System** — CSS variables only, no hardcoded colors
4. **Spacing** — Consistent gaps and padding values
5. **Code Examples** — Real, verified, current versions
6. **Content Quality** — Professional, educational, fact-checked
7. **News Format** — `[Topic - Date] → [summary]`
8. **Images** — SVG preferred, optimized, relative paths
9. **Accessibility** — WCAG compliance, keyboard nav
10. **Performance** — Optimized CSS, minimal JS
11. **Consistency** — All guides match existing style

### Quick Checklist (Pre-Commit Gate)
```
□ HTML semantic and valid
□ Colors use CSS variables only
□ Typography: serif/sans/mono correct
□ Spacing consistent with existing style
□ Links working, relative paths correct
□ Code examples verified with current versions
□ Content tone professional/educational
□ Terminology consistent
□ News: 10 max on homepage, older → archive
□ No hardcoded values, colors, or styles
□ NO Claude/AI mentions anywhere
□ Accessibility standards met
□ Visual consistency with other guides
□ All facts technically accurate
□ Ready to commit
```

## Commit Standards

### ✅ CORRECT Examples
- "Update orchestration guide: clarify routing patterns with latest framework API"
- "Fact-check safety section: fix deprecated validation approach"
- "Add hierarchical agent pattern to advanced section"
- "Organize news: move entries to archive, keep latest 10 on homepage"

### ❌ WRONG (Never Use)
- "Claude updated guides"
- "AI tool added new content"
- "Automated framework version check"
- "Generated by Claude"

**Rule:** Commits must read as if a human wrote them. Zero mentions of AI, Claude, or automation.

## When Agent Skips Changes

The agent will NOT commit if:
1. Update would expand scope beyond agent orchestration
2. Pattern is too speculative or niche
3. Content contradicts existing proven information
4. Facts cannot be verified with high confidence
5. Change fails style guide checklist
